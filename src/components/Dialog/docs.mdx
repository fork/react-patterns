# Dialog

Ein Dialog informiert die Nutzer:innen über einen kritischen Task, der im Vorfeld entschieden oder mitgeteilt werden muss. Ein Dialog kann eine kritische Information beinhalten oder eine Entscheidung einfordern.

Ein Dialog ist ein Modalfenster, dass sich über die aufgerufene Seite legt um kritische Informationen mitzuteilen oder eine Entscheidung einzufordern. Sie deaktivieren jegliche Funktion der Seite bis sie bestätigt oder geschlossen wurden oder die benötigte Entscheidung getroffen wurden ist.

Dialoge unterbrechen die Nutzer:innen absichtlich in ihrem Vorhaben und **sollten deswegen äußerst sparsam eingesetzt werden**.

Dialoge sollten genutzt werden wenn:

- Fehler auftreten, die die "normale Verwendung" der Anwendung verhindern
- Nutzer:innen eine kritische Information bestätigen müssen, eine Entscheidung oder Einwilligung geben müssen

## Anatomie

<!-- Optional: schematische Darstellung -->

1. **Dialogcontainer**: Fläche die Inhalte der Seite verdeckt und Raum für die Nachricht schafft.
1. **Headline**: Die Überschrift sollte eindeutig den Grund der Unterbrechung kommunizieren.
1. **Copytext**: Ergängt ggf. die Headline.
1. **Schließen**: Schaltfläche zum schließen der Nachricht. Kann ausgeblendet werden, wenn eine Entscheidung getroffen werden muss.
1. **Buttons**: Wenn eine Entscheidung getroffen werden muss oder die Nachricht eine Bestätigung verlangt, können die Antwortoptionen als Buttons hinzugefügt werden
1. **Vorhang**: Fläche, die sich über den Seiteninhalt legt und signalisiert, dass diese deaktiviert ist bis der Dialog beantwortet ist.
